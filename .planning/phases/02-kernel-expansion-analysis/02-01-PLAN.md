# Phase 02: Kernel Expansion Analysis - Implementation Plan

The objective of this phase is to implement non-linear kernels (RBF and Polynomial) using the "Kernel Trick" within the dual SVM formulation. We will also implement a comprehensive evaluation script to compare the performance of all three kernels on the breast cancer dataset.

## Wave 1: Kernel Implementation and Registry

### Task 1: Refactor `src/kernels.py` and implement kernels
- **action**: Implement `polynomial_kernel` and `rbf_kernel` in `src/kernels.py` and create a `get_kernel` factory.
- **concrete_values**:
  - `poly_kernel`: $K(x, y) = (\gamma \langle x, y \rangle + r)^d$
  - `rbf_kernel`: $K(x, y) = \exp(-\gamma \|x - y\|^2)$
- **read_first**:
  - [src/kernels.py](file:///src/kernels.py)
  - [.planning/phases/02-kernel-expansion-analysis/02-RESEARCH.md](file:///.planning/phases/02-kernel-expansion-analysis/02-RESEARCH.md)
- **acceptance_criteria**:
  - `src/kernels.py` contains `rbf_kernel` and `polynomial_kernel` functions.
  - `get_kernel` returns the correct function for 'linear', 'rbf', and 'poly'.
  - Vectorized RBF implementation uses the $\|x - y\|^2 = \|x\|^2 + \|y\|^2 - 2\langle x, y \rangle$ identity.

### Task 2: Refactor `src/model.py` for Non-linear Support
- **action**:
  - Update `SVM.__init__` to accept `gamma`, `degree`, and `coef0`.
  - Update `SVM.fit` to handle `gamma='scale'` calculation and use the selected kernel for Gram matrix computation.
  - Refactor `SVM.predict` to support kernel expansion: $f(x) = \text{sign}(\sum \alpha_i y_i K(x_i, x) + b)$.
- **read_first**:
  - [src/model.py](file:///src/model.py)
  - [src/kernels.py](file:///src/kernels.py)
- **acceptance_criteria**:
  - `SVM(kernel='rbf')` can be initialized and fitted without errors.
  - `predict()` method contains a branch for `self.kernel == 'linear'`.
  - Bias `b` is calculated as the mean over free support vectors.
  - `w_` is set to `None` when using non-linear kernels.
